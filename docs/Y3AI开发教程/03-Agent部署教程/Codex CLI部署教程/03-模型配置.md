# 模型配置

安装完 Codex CLI 后，还需要完成模型、API Key 和 API 地址配置。本文推荐优先使用 cc-switch，因为它更适合零基础用户，也方便以后在 Claude Code、Codex、Gemini 等终端 Agent 之间切换。

如果 cc-switch 暂时无法满足你的需求，或者你想理解背后到底写了什么配置，也可以看后面的“手动修改配置文件”分支。

:::tip 推荐顺序
零基础用户建议先走 <strong>cc-switch 配置方式</strong>。手动配置不是更高级，只是更直接；一旦路径、TOML 格式或 Windows / WSL 环境搞错，反而更容易出问题。
:::

## 一、先确认你要配置哪套环境

前面如果选择了 Windows 分支，就配置 Windows 里的 Codex。

前面如果选择了 WSL 分支，就配置 WSL / Ubuntu 里的 Codex。

Windows 和 WSL 是两套独立环境。即使命令都叫 `codex`，实际调用的是哪一套，取决于你当前打开的是 Windows 终端还是 WSL 终端。

## 二、推荐方式：使用 cc-switch 配置模型

如果你使用中转站，或者想统一管理 Claude Code、Codex、Gemini 等终端 Agent 的配置，推荐使用 cc-switch。

cc-switch 项目地址：

https://github.com/farion1231/cc-switch

### 1. 安装 cc-switch

1. 打开 cc-switch GitHub 页面。
2. 进入 `Releases`。
3. 下载最新 Windows 安装包。
4. 双击安装并启动 cc-switch。

### 2. 新增 Codex 配置

不同版本的 cc-switch 界面可能略有差异，但整体思路一致：

1. 打开 cc-switch。
2. 找到 `Codex`、`Provider`、`模型配置` 或类似入口。
3. 点击新增配置。
4. 工具类型选择 `Codex`。
5. 配置名称可以写：

```text
LinkAPI Codex
```

6. API 地址按中转站文档填写。以 OpenAI 兼容接口为例，常见格式类似：

```text
https://api.linkapi.ai/v1
```

7. API Key 填写你的密钥，通常类似：

```text
sk-xxxxxxxxxxxxxxxx
```

8. 模型名称填写中转站支持的 OpenAI / Codex 模型名称。
9. 保存配置。
10. 点击“启用”“切换”或“Apply”，让该配置生效。

如果不确定模型名称，先查看中转站文档或在 cc-switch 中使用模型发现 / 模型列表功能。

### 3. 如果要配置 WSL 里的 Codex，先设置 WSL 配置路径

如果你走的是 Windows 分支，可以跳过这一小节。

如果你走的是 WSL 分支，要先让 Windows 里的 cc-switch 知道 WSL 中 Codex 的配置文件在哪里。否则 cc-switch 可能只会改到 Windows 侧的 `.codex` 配置，而不是 WSL 里的配置。

可以参考这篇外部经验文档：[Windows 下配置针对 WSL 的 cc-switch](https://blog.csdn.net/qq_31805821/article/details/159211262)。

操作步骤：

1. 打开 WSL / Ubuntu 终端。
2. 执行：

```bash
mkdir -p ~/.codex
wslpath -w ~/.codex/config.toml
```

3. 终端会输出一个 Windows 能识别的路径，通常类似：

```text
\\wsl.localhost\Ubuntu\home\你的用户名\.codex\config.toml
```

4. 复制这个路径，但填入 cc-switch 设置时通常要填配置目录，不是具体文件。也就是去掉最后的 `\config.toml`，保留类似下面的目录：

```text
\\wsl.localhost\Ubuntu\home\你的用户名\.codex
```

5. 回到 Windows 中打开 cc-switch。
6. 进入 cc-switch 的设置页面。
7. 找到 Codex 配置目录、配置路径、Config Path 或类似选项。
8. 把上面复制的 WSL `.codex` 目录填进去并保存。

保存后，cc-switch 再切换 Codex 配置时，才会写入 WSL 里的 Codex 配置。

:::warning 注意路径层级
`wslpath -w ~/.codex/config.toml` 输出的是文件路径；cc-switch 设置里如果要求填写“配置目录”，就填到 `.codex` 这一层，不要把 `config.toml` 也填进去。
:::

## 三、备用方式：手动修改配置文件

如果你不使用 cc-switch，也可以手动编辑 Codex 的 `config.toml`。手动配置前请先确认自己当前使用的是 Windows 终端，还是 WSL / Ubuntu 终端。

:::warning 不要把 API Key 发给别人
下面示例里的 `sk-xxxxxxxxxxxxxxxx` 只是占位符。真实 API Key 只应该保存在你自己的电脑里，不要截图发群里，也不要提交到 Git 仓库。
:::

### 1. 先确认配置文件位置

| 你选择的分支 | 配置文件通常在哪里 | 打开方式 |
| --- | --- | --- |
| Windows 分支 | `%USERPROFILE%\.codex\config.toml` | 在 Windows Terminal / PowerShell 中操作 |
| WSL 分支 | `~/.codex/config.toml` | 在 WSL / Ubuntu 终端中操作 |

Windows 和 WSL 是两套独立环境。即使文件都叫 `config.toml`，也不是同一个文件。

### 2. Windows 分支：打开配置文件

在 Windows Terminal / PowerShell 中执行：

```powershell
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.codex"
notepad "$env:USERPROFILE\.codex\config.toml"
```

如果记事本提示“是否创建新文件”，点击“是”。

### 3. WSL 分支：打开配置文件

在 WSL / Ubuntu 终端中执行：

```bash
mkdir -p ~/.codex
nano ~/.codex/config.toml
```

如果你不熟悉 `nano`：

- 粘贴内容后，按 `Ctrl + O` 保存。
- 按回车确认文件名。
- 按 `Ctrl + X` 退出。

### 4. 写入中转站配置

以 OpenAI 兼容接口为例，可以写成下面这样：

```toml
model = "中转站支持的模型名称"
model_provider = "linkapi"

[model_providers.linkapi]
name = "LinkAPI"
base_url = "https://api.linkapi.ai/v1"
env_key = "LINKAPI_API_KEY"
wire_api = "responses"
```

请把：

- `中转站支持的模型名称` 替换成中转站文档里写的模型名称。
- `https://api.linkapi.ai/v1` 替换成你的中转站 API 地址。通常填写到 `/v1` 这一层，不要写成完整的 `/v1/responses` 路径。
- `LINKAPI_API_KEY` 保持为环境变量名称，或者换成你自己想用的环境变量名称。`wire_api` 一般按中转站文档填写；支持 Responses API 时用 `responses`，只支持 Chat Completions 时可能需要改成 `chat`。

:::tip 为什么不直接把 API Key 写进 config.toml？
Codex 的配置文件支持用 `env_key` 指定“从哪个环境变量读取 API Key”。这样配置文件里只保存变量名，不直接保存密钥，更不容易误传或提交到仓库。
:::

### 5. 设置 API Key 环境变量

上一步的 `env_key = "LINKAPI_API_KEY"` 表示 Codex 会从 `LINKAPI_API_KEY` 这个环境变量读取密钥。

Windows PowerShell：

```powershell
$env:LINKAPI_API_KEY="sk-xxxxxxxxxxxxxxxx"
```

WSL / Linux：

```bash
export LINKAPI_API_KEY="sk-xxxxxxxxxxxxxxxx"
```

这种写法只对当前终端窗口生效。关闭终端后，需要重新设置。零基础用户如果觉得麻烦，优先使用 cc-switch。

如果你希望长期生效，可以后续再学习如何设置系统环境变量；不要为了省事把真实 API Key 写进项目文件或截图发给别人。

### 6. 手动配置后重新打开终端

保存 `config.toml` 并设置好环境变量后，建议关闭当前终端窗口，再重新打开一次终端。这样可以避免旧进程继续使用旧配置。

## 四、按你选择的分支打开项目目录

### Windows 分支

推荐用资源管理器打开，避免手动输入路径：

1. 在资源管理器中打开你的 Y3 项目文件夹。
2. 在文件夹空白处点击鼠标右键。
3. 选择 `Open in Terminal`、`在终端中打开` 或 `在此打开终端`。
4. 终端打开后，看一下命令行前面的路径。如果路径就是你的项目文件夹，说明位置正确。

### WSL 分支

更简单的方式是先按 Windows 的方式在项目目录打开终端，然后在这个终端里进入 WSL。

1. 在资源管理器中打开你的 Y3 项目文件夹。
2. 在文件夹空白处点击鼠标右键。
3. 选择 `Open in Terminal`、`在终端中打开` 或 `在此打开终端`。
4. 输入：

```bash
wsl
```

5. 按回车后，如果命令行里出现类似 `用户名@电脑名` 的 Linux 提示符，说明你已经进入 WSL。
6. 输入下面的命令确认位置：

```bash
pwd
```

如果你的项目在 Windows 的 `C:` 盘，路径通常会显示成 `/mnt/c/...`；如果在 `D:` 盘，路径通常会显示成 `/mnt/d/...`。

## 五、启动 Codex 并测试

在项目目录中输入：

```bash
codex
```

进入 Codex 后，先发送最简单的测试消息：

```text
hello
```

如果能正常回复，说明 Codex 已经可以使用。
