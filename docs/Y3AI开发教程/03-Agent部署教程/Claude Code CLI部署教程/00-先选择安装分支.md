# 先选择安装分支

:::tip 验证与时效说明
AI 工具、模型名称、安装步骤和第三方服务变化很快。本文保留实践经验，但涉及时效性的内容，生产使用前请结合官方文档和来源记录再确认。
:::

## 这组教程怎么读

这组文档已经按“先选分支、再执行分支步骤、最后统一验证”的结构拆开：

1. 当前页：先判断自己应该走哪条分支。
2. 二选一：
   - [Windows 分支](./01-Windows分支.md)
   - [WSL 分支](./02-WSL分支.md)
3. 分支安装完成后，再看 [模型配置](./03-模型配置.md)。

你可以同时安装 Windows 和 WSL 两套环境，但不要在同一轮教程里来回混着操作。它们的命令、安装位置和配置文件是分开的；同时维护两套可以做到，但对零基础用户来说更容易混淆。

## 先用这张表做选择

| 你的情况 | 建议选择 | 原因 |
| --- | --- | --- |
| 第一次接触终端 Agent | Windows 分支 | 路径最直观，文件都在 Windows 里，出问题时更容易回到资源管理器检查 |
| 主要做 Y3 项目开发 | Windows 分支 | Y3 编辑器、项目文件、MCP Server 通常都在 Windows 侧 |
| Windows 10 用户 | Windows 分支 | 先安装 Windows Terminal，再按 Windows 分支操作 |
| Windows 11 版本低于 22H2 | Windows 分支 | 不建议走 WSL；WSL 镜像网络可能不可用，影响访问 Windows 本机的 Y3 MCP Server |
| 已经熟悉 Linux / WSL | WSL 分支 | 可以利用更接近 Linux/macOS 的开发环境 |
| 项目明确依赖 Linux 环境 | WSL 分支 | Linux 工具链、脚本和依赖处理通常更顺手 |
| 想深入折腾 omx、omc 等开源 Agent 工具链 | 优先 WSL 分支，但 Windows 也可以用 | 这类工具往往更优先适配 Linux/macOS |

## 两条分支的具体差异

| 选择 | 优点 | 缺点 | 更适合谁 |
| --- | --- | --- | --- |
| Windows 分支 | 安装路径短<br/>文件都在 Windows 里<br/>和 VS Code、Y3 编辑器、资源管理器配合直观<br/>零基础更容易理解 | 个别命令行工具可能更偏向 Linux<br/>遇到依赖编译问题时排查资料相对少 | 大多数 Windows 10 / Windows 11 用户<br/>第一次接触 Agent 的用户<br/>主要做 Y3 项目开发的用户 |
| WSL 分支 | 更接近 Linux 开发环境<br/>很多开源命令行教程可以直接套用<br/>适合复杂工程依赖<br/>omx、omc 等开源 Agent 工具链通常更优先适配 Linux/macOS<br/>部分模型调用工具会优先调用 Linux 环境下的命令 | 多一层 Linux 子系统<br/>Windows 路径和 Linux 路径容易混淆<br/>Node.js、Claude Code、配置文件可能要维护两套 | 已经熟悉 Linux / WSL 的用户<br/>项目明确需要 Linux 环境的用户<br/>愿意处理路径和环境差异的用户<br/>希望折腾更多开源 Agent 工具链的用户 |

## 我的建议

- 如果你是零基础，优先选择 <strong>Windows 分支</strong>。
- 如果你使用 Windows 10，先安装 Windows Terminal，再走 Windows 分支。
- 如果你使用 Windows 11，通常可以直接使用系统自带的 Windows Terminal。
- <strong>如果你的 Windows 11 版本低于 22H2，不建议选择 WSL 分支。</strong>WSL 的镜像网络模式需要 Windows 11 22H2 及以上；低于这个版本时，WSL 里可能无法通过 `127.0.0.1` 直接访问 Windows 本机启动的 Y3 MCP Server。
- 如果你已经熟悉 WSL，或者你的项目强依赖 Linux 环境，可以一开始就选择 <strong>WSL 分支</strong>。
- 如果你后续想深入使用 omx、omc 等开源 Agent 工具链，或者经常使用会调用系统命令的模型工具，WSL 分支会更接近这些工具的默认适配环境。
- 两个分支可以同时安装，但 Windows 和 WSL 的配置文件是相互独立的，需要维护两份配置，会比较麻烦；零基础用户建议先选一条主线跑通。

## 选完分支后不要混用命令

分支选好之后，请按下面的规则理解后续命令：

| 你选择的分支 | 后续“打开终端”指的是 | 安装位置 | 配置位置 |
| --- | --- | --- | --- |
| Windows 分支 | Windows Terminal / PowerShell | Windows 系统里 | Windows 用户目录下的 Claude Code 配置 |
| WSL 分支 | Windows Terminal 里的 Ubuntu / WSL 标签页 | WSL 的 Linux 系统里 | WSL Linux 用户目录下的 Claude Code 配置 |

如果你在 Windows 里安装了一次，又在 WSL 里安装了一次，这是两套独立环境。命令都叫 `claude`，但实际调用的是哪一套，取决于你当前打开的是 Windows 终端还是 WSL 终端。
